# Implementation Notes

The scoped POC deliberately separates two concerns:

- host-side Lean IR extraction using Lean `v4.30.0-rc2`;
- browser-side execution of a wasm32 module that follows the emitted `fib` IR.

`wasm/runner.c` is the intended strict `wasm32-wasi` runner. It is freestanding
and does not require libc headers, but it still needs a WebAssembly linker
(`wasm-ld`) to produce a final `.wasm` file.

`wasm/runner.wat` is a local fallback so the browser demo can run in this
workspace before a WASI SDK is installed. It exports the same ABI as the C
runner and imports `wasi_snapshot_preview1.proc_exit`, but it does not exercise
the C++ upstream interpreter port.

The next porting step is to stage Lean's `src/library/ir_interpreter.cpp` and
the required runtime files from Lean commit
`3dc1a088b6d2d8eafe25a7cd7ec7b58d731bd7cc`, then replace the generated bytecode
runner with a fixture-backed environment provider.
