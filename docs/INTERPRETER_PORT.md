# Upstream Interpreter Port

Milestone 4 is to replace the current generated bytecode runner with Lean's
upstream IR interpreter core for Lean `v4.30.0-rc2`.

## Local Source

Fetch the pinned upstream source into an ignored local directory:

```bash
scripts/fetch-lean-source.sh
```

Audit the interpreter and runtime hotspots:

```bash
scripts/audit-interpreter-port.sh
```

The audit report is written to `build/interpreter-port/audit.md`.

## Initial Port Shape

- Keep the browser ABI stable: `vir_fib`, `vir_target_pointer_bytes`,
  `vir_target_size_t_bytes`, and `vir_target_layout_ok`.
- Compile for `wasm32-wasip1` through the local `.tools/wasi-sdk`.
- Start with `interpreter.prefer_native=false`.
- Disable or stub native symbol lookup, dynamic libraries, libuv, threading, and
  `.olean` loading.
- Provide a fixture-backed declaration provider for the `fib` IR generated by
  Lean 4.30-rc2.

## Current Known Hotspots

- `src/library/ir_interpreter.cpp` uses native symbol lookup for fallback calls.
- The runtime has platform branches for Emscripten but not a dedicated strict
  WASI configuration.
- Full environment/module loading is out of scope for the first port; the first
  port should feed declarations from generated fixtures.
